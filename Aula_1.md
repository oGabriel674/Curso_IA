# Aula 1 - Fundamentos Básicos

## Objetivos
- Entender IA, ML e DL
- Diferenciar aprendizado supervisionado e não supervisionado
- Conhecer aplicações de IA
- Entender o que é um ambiente virtual
- Aprender Git e GitHub

## Inteligência Artificial
IA é o desenvolvimento de sistemas capazes de:
- Aprender com dados
- Identificar padrões
- Tomar decisões automaticamente

> IA = Matemática + Dados + Computação

## Aprendizado Supervisionado vs Não Supervisionado
- **Supervisionado**: usa dados com rótulos, objetivo é prever.  
  Exemplos: preço de casas, spam, fraudes, reconhecimento de imagens.  
- **Não supervisionado**: usa dados sem rótulos, objetivo é descobrir padrões.  
  Exemplos: agrupamento de clientes, recomendação, anomalias, segmentação.

| Característica | Supervisionado | Não Supervisionado |
|----------------|----------------|---------------------|
| Tem rótulo (y)? | ✅ Sim | ❌ Não |
| Existe resposta correta? | ✅ Sim | ❌ Não |
| Objetivo | Prever | Descobrir padrões |
| Exemplo | Classificação | Agrupamento |

## Machine Learning vs Deep Learning
- **ML**: aprende padrões, exige seleção manual de características, funciona bem com datasets menores.  
- **DL**: redes neurais profundas, aprende características automaticamente, exige mais dados e processamento, usado em visão computacional, voz e NLP.

## Inteligência Computacional
Ramo da IA inspirado na natureza e em incertezas:
- **Lógica Fuzzy**: trabalha com graus de pertinência (ex: "0.7 quente").  
- **Algoritmos Genéticos**: inspirados na evolução natural, usados em otimização.  
- **Redes Neurais Artificiais** e outros sistemas biológicos artificiais.

## Ambiente Virtual
- Isola bibliotecas por projeto.  
- Evita conflitos de versões.  
- Facilita reprodução de experimentos.  
- No **Colab** já é automático.  
- No **VSCode**: criar venv, ativar, instalar libs, exportar `requirements.txt`.

## Git e GitHub
- **Git**: ferramenta de controle de versão.  
- **GitHub**: plataforma para armazenar e compartilhar código.  
- Usado para versionar experimentos, colaborar em equipe e criar portfólio.  

### Boas práticas de commit
| Tipo | Uso |
|------|-----|
| feat | Nova funcionalidade |
| fix | Correção de erro |
| refactor | Reorganização |
| docs | Documentação |
| style | Formatação |
| perf | Performance |
| test | Testes |
| chore | Manutenção |
